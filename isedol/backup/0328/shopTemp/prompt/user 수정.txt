이벤트페이지 기능 수정

이벤트 페이지의 각 섹션에 존재하는 grid영역을 수정한다. 그 외 배너 및 분류기능의 3개의 행은 동일하게 유지된다. 다음 내용을 확인하고, 각 섹션의 요구사항을 준수하여 제작하라. SPA형태로 개발해야하며 PC와 모바일 간 UX를 고려하여 동일한 사용자 경험을 제공하라.

다음 내용은 페이지에 존재하는 3개의 섹션 모두 동일하게 적용된다. 또한 그리드 영역 외 슬라이드와 3행의 분류버튼들의 기능 및 디자인은 수정하지 않는다. 다음 선언하는 내용만 수정한다. 또한 다음 영역에 존재하는 기능을 수정하는 내용으로 대체한다.

섹션의 grid영역 수정
1. 그리드 영역에는 2행의 아이템만 출력되며 해당 영역의 크기도 이에 맞춘다.
2. 행의 2줄을 넘어가는 아이템은 출력되지 않는다.
3. 그리드영역 밑에 더보기 버튼이 존재하며 이는 해당 영역의 가운데에 존재한다.
4. 더보기 버튼을 누르는 경우 2행 이후 출력되지 않은 아이템이 출력되며 그리드 영역 높이가 증가한 아이템의 행 수만큼 증가한다.
5. 더보기 버튼을 누른경우 해당 버튼은 닫기로 수정된다.
6. 닫기 버튼을 누르는 경우 처음처럼 그리드 영역에 아이템 2행만큼만 표시되며 외의 아이템은 보이지 않고 버튼도 다시 더보기로 수정된다.

위 내용을 인덱스에 준수하여 수정하되 수정이 필요하지 않은 부분은 유지하도록한다. 수정 후 수정된 부분을 설명하고 전체 내용을 한 파일로 출력하라.

이벤트 페이지의 섹션 내 그리드 영역 수정

이벤트 페이지의 각 섹션에 존재하는 그리드 영역을 수정한다. 다음은 한 섹션의 그리드 영역 수정에 대한 요구사항이다. 각 섹션에 동일하게 적용하며 요구사항에 준수하여 제작하되 다른 페이지의 내용은 일체 수정하지 않는다. SPA형태로 개발해야하며 PC와 모바일 간 UX를 고려하여 동일한 사용자 경험을 제공하라.

그리드 영역 수정 내용
1. 그리드 영역의 스크롤 내용을 제거한다.
2. 초기 아이템 2개의 행만 표시되며 초과되는 아이템은 display를 none으로 설정하여 보이지 않게 한다.
3. 2개의 행 영역 아래 더보기 버튼이 존재한다.
4. 더보기 버튼을 누르는 경우 display가 none으로 설정된 아이템을 보여주며 그리드 영역의 기준에 맞추고 높이를 보여지게 된 아이템의 행 높이만큼 증가시킨다. 이때 버튼은 닫기로 수정된다.
5. 닫기 버튼을 누르는 경우 2번으로 돌아간다.

위 내용을 인덱스에 준수하여 수정하되 수정이 필요하지 않은 부분은 일체 수정하지 않도록한다. 수정 후 수정된 부분을 설명하고 전체 내용을 한 파일로 출력하라.

user페이지 제작

user페이지를 제작한다. user에는 사용자의 기본 정보를 확인 및 수정할 수 있다. 페이지 입장 전 alert를 이용해 비밀번호를 받고 이와 동일하다면 입장하게 된다. SPA형태로 개발해야하며 PC와 모바일 간 UX를 고려하여 동일한 사용자 경험을 제공하라.

사이드 바 버튼의 기능
1. user의 사이드바에 존재하는 버튼을 클릭하는 경우 다음 내용이 실행된다.
2. alert를 이용해 사용자의 비밀번호를 받아 일치하는 경우 user페이지로 이동한다. 현재 정보가 없으므로 비밀번호를 1234로 명시하고 이에 수정할 수 있는 부분을 주석으로 표시한다.

유저페이지
1. 유저의 정보를 수정하는 페이지이다.
2. 유저의 이미지, 닉네임, 등급, 마지막 로그인 날짜, 비밀번호 순으로 작성된다.
3. 유저의 이미지는 단색 카드로 표현된다,
4. 유저 이미지, 닉네임, 비밀번호의 우측에 정보 수정 버튼이 있다. 
4-1. 버튼을 누르는 경우 정보수정 버튼이 사라지고 input폼과 저장버튼이 우측에 생성되며 input폼에 내용이 작성된 후 저장을 누르면 해당 내용의 값이 변경된다, 이 때 공란일경우 수정 없이 기존 4번으로 돌아간다.
4-2. 이미지의 경우 파일을 받아올 수 있는 버튼으로 생성되며 해당 버튼을 눌러 이미지를 가져오는 경우 바로바로 유저 이미지 위치에 표시되고 수정버튼은 저장 버튼으로 변경되 저장을 누르는 경우 유저 이미지가 변경되도록 한다.

위 내용을 인덱스에 준수하여 수정하되 다른 페이지의 js내용 및 css, 관련 코드 모두 수정하지 않도록한다. 위 기능을 제작하는데 필요한 내용만 수정하도록 한다. 수정 후 수정된 부분을 설명하고 전체 내용을 생략없이 한 파일로 출력하라.
