admin 페이지를 제작한다.

 DB와 연결하여 해당 DB의 정보를 가져올 것이다. 슈퍼베이스를 연결할 것이며 Vue로 제작할 것이다. 이때 페이지 전환 방식은 SPA를 따라간다. 페이지의 디자인은 1. 페이지 디자인을 따라갈 것이며 해당 페이지의 기능은 2. 기능을 따라간다.

1. 페이지 디자인
1) 좌측에 고정되고 vh100을 가지는 사이드바가 존재한다. 
2) 이는 db에 있는 테이블을 나타내며 각 테이블 명들은 colunm으로 정렬되어 있으며 클릭하면 우측 내용이 해당 테이블의 row 1번부터 차례대로 출력한다.
3) 한 페이지 내에서 10개까지만 출력되며 10개가 넘어가는 경우 혹은 전체를 해서 pagination을 적용시킨다.
4) 배경화면은 검정색으로 설정한다.

2. 기능
1) 슈퍼베이스를 연결한다.
2) 좌측 사이드바에는 연결된 슈퍼베이스의 테이블 명들을 모아 정렬을 세로로 하여 출력한다.
3) 해당 버튼을 클릭하는 경우 우측에는 해당 페이지에 존재하는 row 들이 pk값에 따라 정렬되어 출력된다,
4) 각 row의 마지막에는 edit버튼과 delete 버튼이 존재한다.
5) 테이블 정보를 출력하는 곳 우측 상단에는 +버튼이 존재한다. 이를 클릭하면 row가 생성된다.

위 인덱스를 준수하여 사이트를 제작한다. SPA방식으로 페이지 전환이 발생할것이며 기능이 동작하도록 사이트를 제작한다. HTML,CSS,JS는 한 파일에 존재하며 전체 코드를 출력해라.

사이드 바의 기능 및 화면 내용을 수정한다. 

사이드바에는 sliders, videoThumbnails, albums, boards, artists, albumKinds, songs, playlists 이름을 가진 버튼들이 존재한다. 이 때 해당 버튼을 클릭하면 우측에 해당 테이블에 존재하는 값들을 가져온다, edit, delete, + 버튼은 그대로 유지한다. html,css,js를 한 파일에 저장하며 파일 형태로 출력한다. 슈퍼베이스 연결 등 사이트바에 존재하는 기능 외 다른 기능은 유지한다.

각 테이블 당 서로 다른 수정페이지를 가진다. 즉 js에서 동적으로 받는 것이 아니라는 것이다. 테이블의 정보를 확인하는 페이지에서 edit버튼을 클릭하면 해당 테이블과 idx의 정보를 가진 후 수정 페이지로 이동하게 된다,

각 수정 페이지에는 각 테이블 당 컬럼 이름과 바로 옆에 현재 저장되어 있는 값들이 저장되며 모든 정보 출력 후 가장 아래에는 save버튼이 존재한다. 이 버튼을 클릭하면 각 컬럼 바로 오른쪽에 저장된 값으로 해당 페이지로 이동할 때 따로 저장한 테이블과 idx값으로 해당 테이블의 idx값을 수정한다.

delete 버튼을 클릭하면 해당 클래스의 해당 idx값을 삭제한다.

+ 버튼을 클릭하는 경우 해당 테이블의 수정페이지와 idx는 현재 테이블의 최고값 +1을 수행하여 테이블과 idx값을 저장한 후 이동한다. 이때는 컬럼에 대한 값이 idx만 존재하기 때문에 해당 값만 input에 저장되어 있고 나머지 컬럼의 우측 입력창은 비워져 있을 때 사용자가 값을 넣고 save 버튼을 클릭하면 해당 row가 추가되도록 만들어라.

페이지 전환은 spa방식으로 전환되어야 하며 현재 존재하는 모든 기능은 유지되어야 한다. html,css,js는 모두 한 파일에 존재한다. 출력은 파일 형태로 출력한다.
