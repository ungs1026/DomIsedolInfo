board 페이지 제작

 해당 페이지의 모든 기능은 width가 1200px 이내에 제작되어야 하며 spa방식으로 페이지를 이동한다. 

 board페이지는 4개의 게시판으로 구성되며 각 게시판은 최대 5개의 게시글만 출력된다. 1. 위치 의 위치에 있다. 각 게시판의 디자인은 2.디자인 으로 제작되어있다. 2.디자인은 한 개의 게시판의 디자인이며 디자인은 모두 동일하다.

1. 위치
1) 총 4개로 구성되어 있으며 한 row에 2개씩 총 2개의 행으로 존재한다.
2) 각 게시판의 제목은 news/ 공지사항/ 자유게시판/ 건의사항이다.
3) 서로 같은 간격을 가지고 있따.
4) 화면 크기가 768px 이하로 내려가는 경우 한 개의 row에 한개씩 총 4개의 row로 정렬 방식이 변경된다.

2. 디자인 
1)  상단과 중앙 하단으로 1: 8 : 1 비율로 영역을 분리한다.
2) 상단에는 좌측 끝에 해당 게시판의 제목과 우측 끝에 +버튼이 존재하며 각각의 제목으로 작성된다.
3) +버튼은 누르는 경우 해당 게시판의 전체 내역이 나오는 페이지로 이동하게된다. 이 때 nav와 playbar는 초기화 및 새로고침이 되지 않아야 한다.
4) 중앙에는 해당 게시판의 게시글로 왼쪽에는 인덱스 변호, 오른쪽에는 게시물의 제목이 나타나며 현재는 정보가 없으니 6개의 더미데이터를 만들도록 한다.
5) 하단은 더보기 버튼으로 이를 클릭하는 경우 해당 게시판의 상단 우측에 존재하는 +버튼을 눌렀을 때와 동일하게 작동한다.

3. 동작
1) 게시판의 우측 상단에 존재하는 +를 클릭하는 경우 클릭한 위치에 존재하는 게시판의 전체 내역을 확인할  수 있는 페이지로 이동된다. 이 항목은 nav에 추가하지 않는다.
2) 해당 게시판으로 이동하면 상단에 게시판 이름과 게시판에 존재하는 모든 게시글을 확인할 수 있으며 페이지네이션이 동작하여 10개가 넘어가면 다음 페이지에서 확인할 수 있다.
3) 게시판의 전체 내용에서 게시글을 클릭하거나 board페이지에서 각 게시판의 게시글을 클릭하는 경우 3.모달 이 생성된다.

3. 모달
1) board페이지 혹은 각 게시판에서 게시글을 클릭하는 순간 생성되며 모달에는 게시글 제목, 다음행에는 작성자와 작성일자, 다음행에 게시글 내용, 다음행에 댓글을 작성할 수 있는 input칸이 존재한다.
2) 우측 상단에는 닫기 버튼이 있으며 닫기 버튼 혹은 모달의 영역 밖을 클릭하면 모달은 꺼지게 된다.
3) 모달에 저장되는 정보는 각 게시판에 존재하는 게시들에 데이터를 저장하이 이 데이터에 따라 모달에 출력되게 한다.

 위 인덱스를 준수하여 페이지를 만든다. 페이지는 spa방식으로 구현할 것이며 html, css, js를 한 파일에 저장한다. 요구사항을 제작하는데 필요한 라이브러리는 cdn 방식으로 가져온다. 디자인은 요구사항을 충족하는 동시에 PC와 모바일간의 UX를 고려하여 구성한다. 수정 혹은 추가가 필요한 코드를 출력하고 어디에 추가해야하는지 출력한다.

다음 사항을 수정 및 추가하고 수정 및 추가된 코드와 어디에 추가하는지 출력한다.

1. 각 게시판의 전체 게시물을 보는 페이지를 각 게시판 당 개별로 존재하게 제작한다. 
2. 전체 게시물을 보는 페이지에 게시물이 10개가 넘어간다면 다음 내용은 페이지네이션을 통해 페이지 번호 클릭 혼은 다음 버튼을 클릭하여 볼 수 있도록한다.
3. board페이지에 존재하는 각 게시판은 게시판별 5개의 개시물만 보이도록 한다.
4. baord페이지에 존재하는 각 게시판에 존재하는 더미데이터를 전체 게시물 페이지에 동일하게 보이도록 한다.
