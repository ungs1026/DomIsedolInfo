playlist페이지 제작

 playlist페이지와 추가로 playlistAll페이지를 만든다. playlist 디자인은 1-1.playlist을 playlistAll은 1-2. playlistAll을 따른다. playlist 기능은 2-1.playlist을 playlistAll은 2-2. playlistAll을 따른다. 현재 데이터가 없으므로 데이터가 필요한 경우 임시 데이터를 이용하며 리스트에서 가져오는 경우 v-for 방식을 이용한다.

페이지 이동
1) playlist페이지에서 playlist페이지 안에 존재하는 grid의 아이템을 클릭하는 경우 playlistAll페이지로 이동한다.

1-1.playlist
1) 현재 저장되어 있는 playlist를 출력한다.
2) 그리드 형식으로 존재한다.
3) 저장된 playlist가 다 출력된다면 마지막은 +버튼이 있는 아이템이 존재한다.

1-2. playlistAll
1) 상단의 nav와 playbar는 동일하게 출력되어야 한다.
2) 상단의 nav아래 대표 이미지를 출력한다.
3) playlist페이지에서 클릭한 아이템에 배열 형태로 존재하는 음악을 board페이지의 detail페이지처럼 출력하는 방법이며 모든 디자인은 board페이지와 분리하여 독립적으로 작성한다.

2-1.playlist
1) 현재 playbar에 존재하는 playlist를 가져온다. playlist에 각 리스트의 id를 의미하는 인덱스와 playlist의 이름을 추가한다. 이를 통해 서로 다른 리스트임을 분리한다.
2) 지금은 1개만 존재하므로 더미 리스트를 2개 더 만들어서 playlist페이지에 총 3개가 출력되도록 한다.
3) 플러스버튼을 누르는 경우 플레이리스트를 추가시키는 모달이 올라가며 이것은 board에서 사용하는 모달과는 개별로 존재한다.
4) playlist페이지에서 grid에 존재하는 아이템을 클릭하면 해당 playlist에 존재하는 노래를 출력하는 playlistAll 페이지로 이동한다.

2-2. playlistAll
1) 상단과 하단으로 이루어지며 상단은 1 하단은 9 비율로 존재한다.
2) 상단에는 좌측에 playlist의 이름, 우측에는 수정버튼을 추가한다.
3) 수정버튼을 누르는 경우 board페이지와 playlist에서 사용하는 모달과는 다른 모달을 추가해 해당 페이지에서 플레이 리스트를 수정할 수 있도록 만든다.
4) 하단에는 playlist페이지에서 클릭한 grid의 아이템에 저장되는 아이디의 playlist 안에 있는 노래를 출력한다.

 위 인덱스를 준수하여 페이지를 만든다. 페이지는 spa방식으로 구현한다. 요구사항을 제작하는데 필요한 라이브러리는 cdn 방식으로 가져온다. 디자인은 요구사항을 충족하는 동시에 PC와 모바일간의 UX를 고려하여 구성한다. 수정 혹은 추가가 필요한 코드를 출력하고 어디에 추가해야하는지 출력한다. 오류가 나거나 수정이 필요한 부분은 추후 다시 요구할 것이다.


전체 코드는 첨부된 파일과 같다.

플레이리스트 페이지의 기능 수정

1. 플레이리스트 페이지의  목록 출력 중 마지막 + 버튼 클릭 시 
1) 해당 기능은 플레이리스트를 추가하는 기능이다.
2) 슈퍼베이스와 연결되어 있으며 이 중 playlists 테이블을 이용한다.
3) 즉 입력값으로 idx는 현재 playlists 테이블에 존재하는 idx 중 최고값 + 1을 플레이리스트 이름은 input으로 thumbnail을 파일 형태로 입력받는다.
4) 현재의 저장 버튼을 save 버튼으로 수정한다.
5) 해당 버튼을 클릭하면 슈퍼베이스와 연결된 상태에서 다음 기능을 수행한다,
 5-1) playlists 테이블에 name에는 플레이리스트이름 저장한다.
 5-2) idx는 테이블에 있는 idx값 중 최대값 + 1을 저장한다.
 5-3) thumbnail은 파일형태로 입력받은 파일을 슈퍼베이스 스토리지에 저장하는데 이 때 테이블명과 동일한 이름으로 존재하는 버킷에 저장한다.

위 인덱스에 준수하여 코드를 수정한다. spa방식으로 페이지가 전환된다. 필요한 라이브러리가 있는 경우 cdn 방식으로 가져온다. html, css, js는 한 파일 안에 존재해야한다. 전체 코드를 출력하지 말고 수정이 필요한 코드만 출력한다. 수정이 필요한 부분 외 수정을 금지한다. 즉 현재 기능은 동일하게 동작해야한다.

플레이리스트전체(playlistall) 페이지의 기능 수정

1. 플레이리스트전체 페이지의  목록 출력 중 수정 버튼 클릭 시 다음 기능이 수행된다.
1) 수정 버튼의 수정을 PlayList Edit으로 바꾼다.
2) selectedPlaylistIdx 즉 플레이리스트에서 플레이리스트전체 페이지로 이동하면서 가져온 selectedPlaylistIdx변수에 저장된 플레이리스트의 idx값을 메인으로 사용한다.
3) playlists 테이블에 존재하는 play_list의 값을 수정하는 기능을 수행하는 버튼이 PlayList Edit 버튼이다.
4) 플레이리스트전체 페이지에서 수정 버튼을 누르면 나오는 모달을 수정한다.
5) 해당 모달의 내용은 상단에 해당 플레이리스트의 name이 출력된다.
6) 다음은 songs 테이블에 존재하는 모든 노래가 목록 형태로 출력된다. 이 때 체크를 할 수 있는 체크박스와 idx와 thumbnail이라는 이미지와 title이 출력되어야 한다.
7) 만약 playlists테이블의 selectedPlaylistIdx 와 같은 idx를 가진 데이터의 play_list에 값이 존재하는 경우 해당 값은 배열로 저장되어 있으며 앞으로도 배열 형태로 저장할 것이다.
8) 값이 있는 경우 이는 songs테이블의 idx로 해당 배열을 하나하나 분리하여 해당 idx와 동일한 songs의 리스트의 체크박스는 체크된 상태여야 한다.
9) 사용자가 체크박스를 통해 play_list에 들어갈 값을 선택한다.
10) 마지막에 save라는 버튼이 존재하며 이 버튼을 클릭하면 체크박스가 체크된 songs의 데이터의 idx를 playlists테이블의 play_list데이터로 배열형태로 수정하여 값을 넣는다.


위 인덱스에 준수하여 코드를 수정한다. spa방식으로 페이지가 전환된다. 필요한 라이브러리가 있는 경우 cdn 방식으로 가져온다. html, css, js는 한 파일 안에 존재해야한다. 전체 코드를 출력하지 말고 수정이 필요한 코드만 출력한다. 수정이 필요한 부분 외 수정을 금지한다. 즉 현재 기능은 동일하게 동작해야한다.

첨부된 파일은 admin 사이트의 전체 코드이다. 

테이블의 각 컬럼을 가져와 수정하는 부분의 기능을 개선한다. 아래 요구사항에서 요구하는 기능에 대해 수정하는 것을 제외한 모든 수정은 금지한다.

songs테이블의 값을 변경하거나 추가할 때 다음 요구사항에 충족하도록 수정 혹은 추가 기능을 수정한다.

1. 수정 페이지
1) mp3file라는 컬럼이 존재할 경우 여기에는 확장자가 mp3인 파일만 받을 수 있으며 mp3를 파일 형태로 받아온다.
2) 슈퍼베이스 스토리지에 테이블 명과 동일한 버킷이 존재한다.
3) 해당 버킷에 mp3파일인 경우 mp3폴더 안에 저장, thumbnail로 저장한 파일은 thumbnail 폴더에 각 값을 저장한다.
4) 각 컬럼의 데이터는 버킷에 저장된 파일들의 url을 가져와서 저장한다.
5) 만일 파일을 수정하는 경우 해당 컬럼의 값이 스토리지에 존재한다면 이를 삭제시키고 새로 추가한 파일을 저장한다.

2. 추가 페이지
1) 행을 추가하는 방식으로 동작한다.
2) thumbnail 컬럼에는 jpg, png, wepg 등 이미지 파일만 파일형태로 받을 수 있다.
3) mp3file 컬럼에는 mp3, flac 파일의 형식만 받을 수 있다.
4) 해당 파일들은 슈퍼베이스의 스토리지에 테이블 명과 동일하게 존재하는 버킷에 저장되며 thumbnail은 thumbnail 폴더 안에 mp3는 mp3폴더 안에 저장된다.
5) thumbnail 컬럼에는 슈퍼베이스 스토리지에 저장된 파일의 url을 mp3file 컬럼의 데이터는 슈퍼베이스 스토리지에 저장된 파일의 url을 저장한다.

나머지 값들에 대한 저장은 현재 기능과 동일하게 작동해야 한다.

위 인덱스의 요구사항을 준수한다. 페이지 전환은 SPA 방식으로 전환되어야 하며 HTML,CSS, JS 모든 내용은 한 파일로 출력되어야 한다. 전체 코드를 생략하지 말고 수정되지 않는 부분까지 모두 포함해서 총 한 파일로 출력해라.
