1. realtime은 테이블의 값 데이터의 변화를 확인하는 방법 중 하나이다.
2. 채널을 생성하며 각 유저마다 그 채널을 구독하여 사용한다.
3. insert나 update 등 값에 변경사항이 있을 때 상황을 판단하여 기능을 실행할 수 있다.

근데 그럼 admin페이지에서는 구독할 필요가 없는거 아니야?
그냥 값을 변경하기만 하면 되는거잖아.
sync테이블의 play만 true / false로 변경하면 되는거잖아
변화를 받아서 기능을 수행하는 경우가 없잖아


현재 매 초마다 확인하면서 변화를 체크하는 방식을 superbase의 realtime 기능으로 수정한다.

1. comment_list와 sync 테이블의 채널을 생성한다.
2. watchparty_room 목록에서 영상을 클릭했을 때 해당 1번에 생성한 채널을 구독한다.
3. 채팅창 출력의 경우에는 어짜피 sync를 눌러 영상을 실행하기 때문에 loadChat함수는 한 번 실행된다.
4. 그 이후부터는 comment_list테이블에 insert가 발생하는 순간마다 같은 watchparty_room_id 값이 현재 접속한 방과 동일한지 확인하고 같다면 이를 채팅창에 추가로 출력한다.
5. sync같은 경우 현재 2초마다 sync테이블에 watchparty_room_idx가 현재 들어온 방의 idx와 비교하여 play컬럼 값이 false인 경우 영상을 정지 true인 경우 재생시키고 있다. 
6. 이를 구독한 sync 테이블에서 play가 update가 발생한 경우를 확인하여 update가 발생했을 때 play값을 받아서 true/false 여부 체크 후 영상 재생, 정지를 판단하도록 한다.

위 인덱스에 맞춰 요구사항에 맞게 코드를 수정한다. 수정이 필요한 부분을 제외하고 코드 수정을 금지한다. 수정 후 전체 코드를 생략없이 파일형태로 출력한다.
